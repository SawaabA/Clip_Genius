Database Design

This database design organizes sports, matches, videos, users, comments, ratings, and tags into separate tables. Each table has a unique ID and links to others using foreign keys. For example, matches link to sports, videos link to matches, and comments/ratings link to videos and users. Tags are assigned to videos through a separate table. The design avoids duplicate data, ensures accuracy, and supports features like adding sports, uploading videos, tagging, commenting, and rating. It is simple, efficient, and scalable.

Database Schema

The database schema consists of the following tables:

  1. Sports: Stores information about sports.

  2. Users: Manages user accounts and roles. Synced With Original Providers Users Database

  3. Matches: Stores details about matches for specific sports.

  4. Videos: Stores video metadata and file paths.

  5. Tags: Manages tags for categorizing videos.

  6. VideoTags: Links videos to tags.

  7. Comments: Stores user comments on videos.

  8. Ratings: Stores user ratings for videos.
